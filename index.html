HTML


  • The head element contains metadata about the webpage.
  • The body element contains the visible content that's shown to the user.
  • Child elements are elements that are nested within other elements.
  • Parent elements are elements which contain nested children.

CSS


  • Inline CSS: used to style an element directly in the HTML
  • Internal Style Sheet: contains CSS rules for the webpage in the head of the HTML
  • External Style Sheet: links the style sheet using a link element placed in the head
  • Declaration: contains the CSS property we want to apply (ie: color) along with the value of that property (ie: blue)
  • Margin: The margin property indicates how much space is around the outside of an element.
  • Padding: The padding property adds space around the content within an element.

Git


  • Repository: A folder for your project which contains all project files as well as the version history
  • Branch: A workspace where you can make changes to the code without affecting the main branch or live site
  • Commit: Saving a new version of a file you're working on
  • Pull Request: Asking for changes on the current branch to merge into another branch (usually the main branch)
  • Merge: When a pull request is approved, changes made in your branch are "pulled in" or merged into the main branch.
  • Issue: A way of reporing new tasks or content fixes
  • git status: checks what branch we're currently on
  • git checkout -b branch-name: creates a new branch and switches over to it
  • git add -A: stages all files and changes in the current branch to commit
  • git commit -m: commits all staged files and changes in the current branch. "-m" allows us to add a message
  • git pull origin main: syncs our local repo up with the remote repo
  • git push origin branch-name: pushes all changes on current branch up to the remote repo to ready it for a pull request
